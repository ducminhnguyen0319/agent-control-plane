# Code of Conduct

## Our Goal

This project should feel technically sharp, respectful, and safe for people
who want to build, debug, review, and operate agent workflows together.

## Expected Behavior

Participants in issues, pull requests, reviews, and other project spaces are
expected to:

- be respectful, clear, and constructive
- criticize ideas, code, and behavior without attacking people
- assume good intent until there is evidence otherwise
- share feedback in a way that helps the project move forward
- respect boundaries, privacy, and security concerns

## Unacceptable Behavior

The following are not acceptable in project spaces:

- harassment, intimidation, or threats
- hateful, discriminatory, or degrading language
- sexualized language or unwanted sexual attention
- doxxing, stalking, or publishing private information without permission
- trolling, baiting, personal attacks, or sustained bad-faith disruption
- spam, scams, or malicious social engineering

## Scope

This Code of Conduct applies to repository spaces such as issues, pull
requests, reviews, and discussions, as well as other project-managed channels
where people are representing or participating in this project.

## Enforcement

The maintainer may take any action needed to protect the project and its
participants, including:

- asking for a change in behavior
- editing, hiding, locking, or removing content
- closing discussions or pull requests
- temporarily or permanently restricting participation

## Reporting

If you experience or witness behavior that violates this Code of Conduct,
contact the maintainer privately through `https://github.com/ducminhnguyen0319`.

For undisclosed security issues, use [SECURITY.md](./SECURITY.md) instead of a
public report.
